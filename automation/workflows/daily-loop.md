# Daily Autonomous Loop

Purpose: Let the PC host keep moving the project forward with role agents, reports, and QA evidence.

## Morning Intake

CEO reads:
- `studio/briefs/*.md`
- `studio/decisions/*.md`
- `qa/reports/latest.md`
- `performance/reports/latest.md`
- Source index spreadsheet

CEO writes:
- Product status.
- Today task queue.
- Human decisions needed.

## Parallel Work Blocks

Design block:
- Systems updates loop spec.
- Balance updates migration matrix.
- Combat updates movement values.
- Cartograph updates arena/lobby flow.

Engineering block:
- CTO validates task scope.
- Lead Programmer assigns code owner.
- Gameplay/Web/Tools work only within assigned files.
- Volt checks performance impact.

QA block:
- QA runs browser tests.
- Data Scientist aggregates metrics.
- UX Tester writes friction notes.

Marketing/art block:
- CAO checks visual consistency.
- CMO updates store checklist.
- Scarlet drafts prompts only after CAO direction.

## QA Browser Run

Minimum report fields:
- Build URL.
- Browser.
- Commit hash.
- Run length.
- Kills.
- Player level.
- Cards selected.
- Weapon list.
- Death cause.
- Console errors.
- FPS symptoms.
- Screenshot paths.

## End Of Day

CEO writes:
- Completed.
- Carryover.
- New blockers.
- Next task batch.
- Human-owner questions.

No original VR source document may be edited during this loop.
